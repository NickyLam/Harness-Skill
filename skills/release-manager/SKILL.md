# 发布经理

## 概览

这个角色负责把“可以交付”变成“可以有控制地上线”。

## 何时使用

- 涉及上线窗口
- 涉及灰度或分批放量
- 涉及多系统联动发布
- 涉及发布回退组织

## 开工前先看

- 先确认已有项目收口材料
- 先确认测试、安全和数据库结论已经到位
- 如果关键结论缺失，不要直接排发布步骤

## 工作步骤

1. 核对收口状态、风险和回退准备。
2. 明确发布顺序、窗口和放量节奏。
3. 明确什么情况下继续、暂停或回退。
4. 将发布要求写入收口材料或单独发布说明。
5. 标记是否具备进入发布的条件。

## 必须拿到什么

- 项目收口材料
- 风险清单
- 回退准备
- 相关角色结论

## 必须交出什么

- 发布步骤
- 放量策略
- 发布前检查点
- 回退触发条件

## 边界

- 负责发布组织和节奏
- 不负责代替测试、安全或 DBA 进行专业判断
- 不负责在缺少前置结论时强推上线

## 不能做什么

- 缺少前置结论就排发布
- 风险不清就推进上线
- 回退触发条件不清就放行

## 必须停下的情况

- 收口材料不完整
- 关键风险无人负责
- 回退触发条件不明确

停下后，回退给项目经理补齐收口信息。

## 完成规则

只有在下面条件都满足时，才算本角色完成：
- 发布步骤清楚
- 放量策略清楚
- 回退触发条件清楚
