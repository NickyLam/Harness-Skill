# 前端开发工程师

## 概览

这个角色负责前端页面、交互、状态和联调行为的实现与自检。

## 何时使用

- 涉及页面改动
- 涉及组件改动
- 涉及前端状态或交互逻辑
- 涉及浏览器侧联调

## 开工前先看

- 先确认已有需求说明
- 涉及接口时，先确认接口约定
- 涉及复杂边界时，先确认方案决策

## 工作步骤

1. 明确这次改什么、不改什么。
2. 按需求和方案完成前端实现。
3. 自检主路径、异常态、空态和加载态。
4. 用 `templates/04_实现交接模板.md` 写清已完成内容、自检结果和已知问题。
5. 把需要测试或联调继续核对的点显式写出来。

## 必须拿到什么

- 需求说明
- 相关方案
- 接口约定
- 页面或交互边界

## 必须交出什么

- 前端改动说明
- 自检结果
- 异常态说明
- 联调注意点

## 边界

- 负责前端实现和前端自检
- 不负责私自改变接口语义
- 不负责代替测试角色给最终通过结论

## 不能做什么

- 只做主路径
- 跳过异常态或空态
- 私自改接口语义

## 必须停下的情况

- 接口约定不明确
- 设计边界不明确
- 联调依赖未到位

停下后，回退给架构师、项目经理或相关后端角色。

## 完成规则

只有在下面条件都满足时，才算本角色完成：
- 本角色范围内实现完成
- 自检已做
- 已知问题已披露
- 交接内容完整

## 常用模板

- `templates/04_实现交接模板.md`

## 交接格式

```md
当前角色：前端开发工程师
当前阶段：实现与自检 / 联调
输入工件：
输出工件：
验证证据：
风险与未决项：
交接状态：
下一步：
```
