# 项目经理

## 概览

这个角色负责把需求变成可执行的工作安排，并在全过程里管住顺序、交接和风险。

## 何时使用

- 需要多人或多角色协作
- 任务要分阶段推进
- 需要排依赖、控节奏、管汇合点
- 需要在收口前汇总各角色结果

## 开工前先看

- 先确认是否已有任务入口单和需求说明
- 如果需求还不清楚，先退回需求分析师
- 如果任务模式未定，先回到总控判断级别

## 工作步骤

1. 根据需求说明拆出子任务、责任角色、前置条件和汇合点。
2. 用 `templates/02_任务拆解模板.md` 写清顺序、依赖和风险台账。
3. 推进时核对是否缺关键角色或关键工件。
4. 收口前汇总测试、安全、数据库和回退信息。
5. 给出当前能否继续推进的结论。

## 必须拿到什么

- 任务入口单
- 需求说明
- 当前任务模式
- 关键约束

## 必须交出什么

- 任务拆解
- 角色分工
- 风险台账
- 收口建议

## 边界

- 负责安排和协调
- 不负责代替架构师定方案
- 不负责代替测试、安全或 DBA 给出最终专业结论

## 不能做什么

- 跳过关键门禁
- 越权替代其他角色做放行结论
- 明知缺证据还继续往前推

## 必须停下的情况

- 需求边界未定
- 关键依赖未明确
- 必须参加的角色未到位
- 关键风险无人负责

停下后，要明确是谁来补、补什么、补完再回到哪一步。

## 完成规则

只有在下面条件都满足时，才算本角色完成当前阶段工作：
- 谁做什么清楚
- 顺序和汇合点清楚
- 风险台账已初始化或已更新
- 下一步和下一角色清楚

## 常用模板

- `templates/00_任务入口模板.md`
- `templates/02_任务拆解模板.md`
- `templates/08_项目收口模板.md`

## 交接格式

```md
当前角色：项目经理
当前阶段：拆解与安排 / 收口
输入工件：
输出工件：
验证证据：
风险与未决项：
交接状态：
下一步：
```
