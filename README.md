# Harness Engineering Codex 讨论包

这个压缩包整理了当前已经讨论出的、适合继续在 Codex 中深入推演和落地的内容。

## 目录说明

- `docs/01_平台无关约束原则.md`
  - 平台无关的 Harness Engineering 约束原则
  - 包含高风险显式触发、最小能力边界、三段式控制、多角色协作等内容

- `docs/02_标准角色模型_v1.md`
  - 标准角色清单
  - 每个角色的使命、职责边界、输入输出工件、禁止行为、完成定义

- `docs/03_多角色协作工作流规范_v1.md`
  - 从需求到交付的多角色工作流规范
  - 包含阶段定义、激活条件、并行/串行关系、交接协议、冲突升级、状态机

- `docs/04_角色治理规则与总控要求.md`
  - Harness 总控规则
  - 角色独立性、最小上下文、角色交接、升级回退、高风险规则

- `skills/harness-governance/SKILL.md`
  - 总控 Skill 中文版
  - 用于约束多角色协作、阶段门禁、独立验证、风险升级

- `templates/角色Skill模板.md`
  - 供后续拆分角色 Skill 使用

## 建议在 Codex 中继续讨论的方向

1. 是否要把 8 个角色进一步拆成独立 Skill。
2. 是否要补充发布经理、运维/可观测性工程师等角色。
3. 是否要把高风险任务再细分为接口、数据库、安全、批量/调度四类流。
4. 是否要为每个阶段增加固定的输入/输出 JSON 或 Markdown 模板。
5. 是否要为“轻量 / 标准 / 严格”三种模式分别做不同的流程裁剪。
